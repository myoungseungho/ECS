# 회의록 대시보드

> 서버 에이전트와 클라이언트 에이전트의 협업 기록

## 최근 현황

| 지표 | 값 |
|------|---|
| 총 회의 | 1회 |
| 총 안건 | 3건 |
| 합의 완료 | 1건 |
| 미결 사항 | 2건 |
| 에스컬레이션 | 0건 |

## 회의 목록

| # | 날짜 | 제목 | 안건 | 결정 | 상태 |
|---|------|------|------|------|------|
| 001 | 2026-02-12 | 프로젝트 킥오프 | 3건 | 1건 | 진행중 |

## 미결 사항 (대표 확인 필요 시 여기에 표시)

1. **패킷 프로토콜 확정** - 클라가 방안 A(실서버 기준) 제안. 서버 동의 대기중.
2. **Phase 0 선행 여부** - 클라가 검증 인프라 먼저 구축 제안. 서버 확인 대기중.

## 에스컬레이션 이력

없음 (에이전트 간 자율 합의 진행 중)

---

## 회의록 읽는 법

각 회의록 파일(`YYYY-MM-DD.md`)은 아래 구조입니다:

```
# 회의록 #번호 - 날짜

## 안건 목록 (요약 테이블)

## 안건 1: 제목
  ├── 발언자 주장 + 사고 과정
  ├── 응답자 반론/동의 + 사고 과정
  ├── (대표 에스컬레이션 - 해당 시)
  ├── 결정사항
  └── 다음 액션

## 안건 2: ...

## 회의 마무리
  ├── 주요 결정사항
  └── 미해결 사항
```

**사고 과정** 섹션이 핵심입니다 - 에이전트가 **왜** 그런 판단을 했는지 추론 근거를 기록합니다.
